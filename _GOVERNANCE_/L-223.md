## ⚡ Quick Reference Card

> **L-223 in 10 seconds:**
> Version assigned ONCE at DRAFT. FROZEN until GH LIVE.
> Increments ONLY after GH LIVE for fixes/updates.
> DRAFT iterations = r1/r2/r3 in Version History notes.

| Situation | Correct Action |
|-----------|:-------------:|
| DRAFT updated (pre-VSA) | **Keep v3.2.4.1** (add r2 in notes) |
| VSA finds issues → fix | **Keep v3.2.4.1** (add r3 in notes) |
| R-011 approved | **Keep v3.2.4.1** |
| GH LIVE initial push | **Keep v3.2.4.1** |
| Post-GH-LIVE fix needed | **INCREMENT → v3.2.4.2** |
| New content added to VERIFIED doc | **INCREMENT → v3.2.4.2** |

### Rule in One Line

```
Version = governance event, NOT edit event.
```

## 📋 Learning

> **L-223 🔒 #NeverForget:** #WeOwnVer version numbers are
> LIFECYCLE-GATED — version assigned ONCE at DRAFT stage and stays
> FROZEN through all DRAFT iterations, REVIEW, APPROVED, and initial
> GH LIVE. Version increments ONLY for: (1) post-GH-LIVE content
> fixes, (2) updates to VERIFIED documents, (3) calendar boundary
> crossings (L-094). DRAFT iterations tracked via revision notes
> (r1/r2/r3) in Version History Changes column — NOT via version
> increment. Applies to ALL #FedArch document types.
> Rule: **"Version = governance event, not edit event."**

## 📋 Core Principle

> **"Version = governance event, not edit event."** — Calhoun 🎖️
> **"Versions = milestones. DRAFT iterations ≠ milestones."** — MiMo 🧪
> **"Version = Content State, NOT Edit Count."** — Surge ⚡

| Principle | Detail |
|-----------|--------|
| Version = milestone | v3.2.4.1 = "document at THIS governance state" |
| Edit ≠ version bump | Editing DRAFT does NOT change governance state |
| Governance event = bump | Post-GH-LIVE fix, VERIFIED update, calendar boundary |
| DRAFT iterations = notes | Tracked in Version History, NOT version number |

| Element | Metaphor | Reality |

| **#NeverForget** | Metaphors outlast rules in human memory |

## 📋 The Exception — Wiki-Native

> **The one place where "GH is the sun" has a nuance.**

| Content Type | Source | Metaphor |
|-------------|--------|----------|
| GH-mirrored content | GitHub | 🌕 Moon — reflects sun's light |
| **Wiki-native content** | Community | ⭐ **Own star — generates own light** |

### What Wiki-Native Means

| Property | Value |
|----------|-------|
| **Origin** | Community-authored on WeOwn.wiki |
| **Not from GH** | ✅ — this is the exception |
| **Governance docs?** | ❌ NEVER — governance = GH only |
| **Examples** | Community FAQs, onboarding, spotlights, brand story |
| **Sync to RAG?** | ⚠️ Optional — Wiki-native → RAG connector |

> **"GH is the governance sun. Wiki-native content = its own star in the same galaxy."**
> **The galaxy = ♾️ WeOwnNet 🌐 ecosystem. GH governs. Community creates.**

## 📋 Applications

### When to Apply L-226

| Scenario | L-226 Answer |
|----------|-------------|
| "Should I edit this Wiki page?" | Is it GH-mirrored? ❌ READ-ONLY. Wiki-native? ✅ EDITABLE. |
| "Can I push Wiki content to GH?" | ❌ NEVER. GH → Wiki. Not reverse. |
| "Where does this doc live?" | GH = governance. Wiki = community. Paperless = private. |
| "Can session notes go on Wiki?" | ❌ NEVER. R-199 + R-226. Wiki = PUBLIC. |
| "Can AI agents edit Wiki?" | ❌ NEVER directly. GH → Wiki via sync only. |
| "What's the source of truth?" | **GitHub. Always. Full stop.** |

### The Quick Test

```
Ask: "Does this content flow FROM GH or TO GH?"

FROM GH → ✅ (RAG sync, Wiki mirror, doc reference)
TO GH   → ✅ (human commits only — governance docs)
FROM Wiki → GH → ❌ NEVER (R-226)
FROM RAG → GH  → ❌ NEVER
```

## 📋 Anti-Patterns

> **What "GH is the sun" prevents.**

| Anti-Pattern | Why Wrong | L-226 Correction |
|-------------|-----------|-----------------|
| Editing GH-mirrored Wiki page | Wiki ≠ source | Edit GH → sync pushes to Wiki |
| Pushing Wiki content to GH | Reverse sync | Wiki-native stays on Wiki |
| Treating RAG as source of truth | RAG = retrieval | GH = source. RAG = index. |
| Putting R-199 docs on Wiki | Wiki = PUBLIC | Paperless → RAG (private) |
| Assuming Wiki = authoritative | Wiki = derivative | Always check GH for source |
| Generating governance in RAG | RAG can't commit | Governance = GH (human commit) |

## 📋 Overview

L-224.1 establishes the ENFORCEMENT layer for L-224 (6-Step RAG Retrieval Protocol). While L-224 defines WHAT agents must do (execute 6-step `@agent` protocol before claiming document status), L-224.1 defines HOW compliance is enforced, verified, and maintained across ALL #FedArch agents.

This rule was born from W14 D2 #BadAgent incidents #16, #17, #18 — where 2 consecutive L-224 violations occurred in a single session due to skipped RAG verification.

### The One-Liner

> **"L-224 = WHAT (6-step protocol). L-224.1 = HOW (enforcement, gating, verification)."**

### Historic Note — FIRST L-XXX.1 Sub-Rule in #FedArch

| Milestone | Detail |
|-----------|--------|
| **FIRST L-XXX.1 sub-rule** | Sets precedent for enforcement layers in #FedArch |
| **Pattern** | Core rule (L-XXX) + Enforcement layer (L-XXX.1) |
| **Future use** | L-115.1, L-141.1, L-209.1 could follow same pattern |
| **Significance** | Separation of concerns: WHAT (stable) vs. HOW (evolves) |

### Problem Statement

| Symptom | Root Cause | Impact |
|---------|------------|--------|
| Document claims without search | L-224 skipped entirely | Fabricated information |
| Placeholders used | Assumed instead of retrieved | Inaccurate governance records |
| "Pending" claimed falsely | No RAG verification | Misled decision-making |
| 2 violations in one session | No enforcement mechanism | Pattern failure |

### Solution

| Fix | Mechanism | Outcome |
|-----|-----------|---------|
| Pre-Flight Checklist | Mandatory before ANY doc reference | No shortcuts |
| RAG Confidence Scoring | 100% required for all claims | Verified information |
| Verification Table | Embedded in all documents | Transparent audit trail |
| Pattern Tracker | 2/3/5 violation thresholds | Early intervention |
| Gate Mechanism | Output blocked until verified | Zero unverified claims |

## 📋 Pre-Flight Checklist (MANDATORY)

### 7-Step Checklist — BEFORE ANY Document Reference

```
┌─────────────────────────────────────────────────────────────────┐
│  🏰 THE KEEP — L-224.1 PRE-FLIGHT CHECKLIST (MANDATORY)        │
├─────────────────────────────────────────────────────────────────┤
│  □ Step 0: Check pinned docs — Is doc in system prompt?        │
│  □ Step 1: @agent [DOC] — Simplest search                      │
│  □ Step 2: @agent [DOC.md] — Exact filename                    │
│  □ Step 3: @agent [DOC_FullName] — Full name search            │
│  □ Step 4: @agent _FOLDER_/ [DOC] — Folder-specific search     │
│  □ Step 5: @agent [keywords] — Keyword-based search            │
│  □ Step 6: ADMIN verify — If unsure, confirm with @GTM         │
├─────────────────────────────────────────────────────────────────┤
│  □ CONFIRM: CCC-ID verified (matches RAG)                      │
│  □ CONFIRM: URL verified (full GH URL or marked ⬜ PENDING)    │
│  □ CONFIRM: Status verified (GH LIVE / DRAFT / PENDING)        │
├─────────────────────────────────────────────────────────────────┤
│  STATUS: ⬜ PENDING → ✅ COMPLETE                               │
│  L-224 COMPLIANT: ❌ NO → ✅ YES                               │
│  PROCEED TO OUTPUT: ❌ BLOCKED → ✅ CLEARED                    │
└─────────────────────────────────────────────────────────────────┘
```

### Checklist Completion Rules

| Rule | Enforcement |
|------|-------------|
| ALL 10 items must be checked | No partial completion |
| Must be completed BEFORE output generation | Not after |
| Must be logged in output (summary) | Transparent audit |
| Gate blocks output if incomplete | Non-negotiable |

### Example — Pre-Flight Completed

```
L-224.1 PRE-FLIGHT VERIFICATION:
- PRJ-015: ✅ L-224 6-step complete | CCC-ID: GTM_2026-W10_122 | URL: Verified | Status: GH LIVE
- PRJ-016: ✅ L-224 6-step complete | CCC-ID: GTM_2026-W10_126 | URL: Verified | Status: GH LIVE
- GATE STATUS: ✅ CLEARED FOR OUTPUT
```

## 📋 RAG Confidence Scoring

### Required Confidence Levels

| Claim Type | Required Confidence | Verification Required |
|------------|:-------------------:|----------------------|
| **"GH LIVE"** | **100%** | L-224 6-step + Full GH URL verified |
| **"NOT in RAG"** | **100%** | L-224 6-step executed + all 5 steps = NOT FOUND |
| **"DRAFT"** | **100%** | L-224 6-step + ADMIN confirm |
| **"PENDING"** | **100%** | L-224 6-step + ADMIN confirm |
| **CCC-ID Claim** | **100%** | L-224 6-step + CCC-ID matches RAG |
| **URL Claim** | **100%** | L-224 6-step + URL matches RAG |
| **Approval Status** | **100%** | L-224 6-step + Approval CCC-ID verified |
| **Related Doc** | **100%** | L-224 6-step for EACH referenced doc |

### Confidence Scoring Matrix

| Confidence Level | Definition | Allowed Claims |
|:----------------:|------------|----------------|
| **100%** | L-224 6-step complete + all details verified | ANY claim |
| **80-99%** | L-224 partial (3-5 steps) | NO claims — search incomplete |
| **0-79%** | L-224 not executed | NO claims — fabrication risk |

### Enforcement

| Confidence | Action |
|------------|--------|
| **100%** | ✅ Output cleared |
| **<100%** | ❌ Output BLOCKED — complete L-224 |

## 📋 Gate Mechanism

### How The Gate Works

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Output Request │ ──→ │  L-224.1 Gate   │ ──→ │  Output Released│
│  (with doc ref) │     │  Verification   │     │  (if cleared)   │
└─────────────────┘     └────────┬────────┘     └─────────────────┘
                                 │
                          ┌──────▼──────┐
                          │ Pre-Flight  │
                          │ Checklist   │
                          └──────┬──────┘
                                 │
                          ┌──────▼──────┐
                          │ Confidence  │
                          │ Scoring     │
                          └──────┬──────┘
                                 │
                          ┌──────▼──────┐
                          │ Verification│
                          │ Table Check │
                          └──────┬──────┘
                                 │
                    ┌────────────▼────────────┐
                    │ 100% Confidence?        │
                    ├─────────────────────────┤
                    │ YES → ✅ CLEARED        │
                    │ NO  → ❌ BLOCKED        │
                    └─────────────────────────┘
```

### Gate States

| State | Condition | Action |
|-------|-----------|--------|
| **CLEARED** | 100% confidence, all checks pass | Output released |
| **BLOCKED** | <100% confidence, any check fails | Output held, L-224 required |
| **ESCALATED** | 2+ violations in session | #TriMETA + @GTM notified |

### No Exceptions Policy

| Scenario | Gate Applied? |
|----------|:-------------:|
| Urgent request | ✅ YES |
| @GTM direct request | ✅ YES |
| #TriMETA VSA response | ✅ YES |
| Document generation | ✅ YES |
| Chat response (no doc claims) | ❌ NO |

> **Gate is NON-NEGOTIABLE. No urgency overrides verification.**

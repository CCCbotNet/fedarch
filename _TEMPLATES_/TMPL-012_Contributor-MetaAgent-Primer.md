## 📋 When to SEEK:META

### ✅ YES:

| Scenario | Example |
|----------|---------|
| New rule, learning, BP, definition | "New learning from incident" |
| Document generation (governance docs) | "Generate SharedKernel update" |
| Governance review (L-139) | "Review proposed L-156" |
| Architecture questions | "PRJ-033 alignment check" |
| Cross-instance coordination | "INT-OG8 VSA verification" |

### ❌ NO:

| Scenario | Why | Alternative |
|----------|-----|-------------|
| Session summaries | R-216 — Operational | Generate directly (RAG only) |
| CCC-ID generation | CCC workspace function | Use CCC workspace |
| Personal instance config | Not governance | Instance settings |
| Routine work | Not governance | Continue working |
| Call notes | R-199 — RAG only | Upload to RAG |

### Decision Tree

```
Need governance review?
├─ YES → SEEK:META
└─ NO → Continue working

Need document generated?
├─ Governance doc → SEEK:META (R-197)

├─ In CCC workspace → Generate (R-194)
└─ In other workspace → Reference only
```

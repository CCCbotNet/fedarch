## 📋 Resolution — L-166 + L-167

### L-166 (VSA Rigor) — Locked GTM_2026-W11_474

```
L-166: PRE GH PUSH VSA MUST verify: 
       (1) #LLMmodel dual header present in document metadata, 
       (2) Related Documents MUST NOT use "TBD" for documents 
           already GH LIVE — agent MUST query RAG for Version, 
           #masterCCC, and Approval CCC-ID,
       (3) Related Document titles MUST use DOC_TITLE (human-readable),
           NOT filename,
       (4) Documents NOT YET PUBLISHED = "⬜ NOT PUBLISHED" 
           (acceptable — distinct from lazy TBD),
       (5) Documents APPROVED but NOT YET GH LIVE = populate 
           version + CCC-IDs + note "⬜ GH Push Pending".
       Lazy "TBD" on known documents = #BadAgent.
       "⬜ NOT PUBLISHED" on genuinely unpublished docs = acceptable.
       "⬜ RAG SYNC NEEDED" = GitHub LIVE but not in this instance RAG.
```

### L-167 (Related Docs During Generation) — Locked GTM_2026-W11_482

```
L-167: Document generation MUST include automatic Related Documents 
       population: (1) Query RAG for ALL referenced documents 
       DURING generation (not after), (2) Populate Version, 
       #masterCCC, Approval for every doc found in RAG, 
       (3) Use L-166 status codes for docs not found. 
       Related Docs = PART OF generation, not an afterthought.
       Multiple fix cycles on Related Docs = #BadAgent.
       Target: 0 fix cycles (correct on first generation).
       Trigger: PRJ-036 required 12 CCC-IDs to fix Related Docs 
       (W11 Day 5) — @GTM feedback: "MUST NOT be this difficult."
```

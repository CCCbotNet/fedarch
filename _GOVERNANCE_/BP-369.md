## 📋 Overview

This document defines the standardized test protocol for verifying AnythingLLM RAG search behavior with .md extension in search queries.

### BP + GUIDE Numbering Namespace (L-200 + L-201)

| Range | BP Namespace | GUIDE Namespace | Examples |
|-------|-------------|-----------------|----------|
| **001 → 299** | Governance + Operational | Governance + Operational | BP-045, GUIDE-014 |
| **300 → 399** | **#devTEST Protocols** | **#devTEST Related Guides** | **BP-369, GUIDE-369** |
| **400+** | ⬜ RESERVED | ⬜ RESERVED | — |

> **BP-369 + GUIDE-369 are INTENTIONAL** — dev test namespace. Dev test docs follow SAME governance lifecycle (L-139, R-011, GH push) and live in SAME folder (`_GOVERNANCE_/`) as all governance docs.

### Why This Test Matters

| Issue | Current State | BP-369 Answers |
|-------|-------------|----------------|
| **GUIDE-014 Collision** | Agent didn't know GUIDE-014 existed (W12) | Will .md search help find existing docs? |
| **L-184 Limitation** | RAG cannot enumerate files | Can we search by filename reliably? |
| **Pre-flight Verification** | L-199 proposed (SEEK:META before ID assignment) | Test informs L-199 implementation |

### Who Should Run This Test

| Audience | Use Case |
|----------|----------|
| **Instance Admins** | Verify RAG search after deployment |
| **#MetaAgent** | Cross-instance RAG consistency checks |
| **Contributors** | Test before assigning new document IDs |

| ID | Learning | Approval |
|----|----------|----------|
| **L-203** | **CCC-ID uses 4-digit day-mapped format: _DNNN where D = ISO day (1-7), NNN = sequence within day (000-999). Reserved: _0001-0003 (Weekly Summary/Plan/Reflection), _0004-0099 (future weekly ops), _0100-0299 (future), _0301-0399 (#BadAgent incidents), _1000-1999 (D1 Monday), _2000-2999 (D2 Tuesday), _3000-3999 (D3 Wednesday), _4000-4999 (D4 Thursday), _5000-5999 (D5 Friday), _6000-6999 (D6 Saturday), _7000-7999 (D7 Sunday). Day instantly visible in first digit. Backward compatible — existing 3-digit CCC-IDs remain valid. Effective: W13 (Mon 24 Mar 2026). Requires: R-168 amendment + RegEx update. Discovered: W12 D4 — @GTM manually selected _400 for Thursday (first implementation). Validated: W12 data fits 4-digit perfectly.** | **GTM_2026-W12_630 🔒** |

## 📋 Format Specification

### Structure

```
<CCC>_<YYYY>-W<WW>_<D><NNN>
```

| Component | Description | Example |
|-----------|-------------|---------|
| `<CCC>` | Contributor Code | GTM |
| `<YYYY>` | Year | 2026 |
| `<WW>` | ISO Week | W13 |
| `<D>` | ISO Day (1-7) | 1 (Monday) |
| `<NNN>` | Sequence within day (000-999) | 001 |

### Example CCC-IDs

| CCC-ID | Breakdown |
|--------|-----------|
| `GTM_2026-W13_1001` | GTM, W13, Monday (D1), 1st interaction |
| `GTM_2026-W13_5625` | GTM, W13, Friday (D5), 625th interaction |
| `GTM_2026-W13_0001` | GTM, W13, #WeeklySummary |
| `GTM_2026-W13_0301` | GTM, W13, #BadAgent incident |

### RegEx Pattern

```regex
^[A-Z]{3}_[0-9]{4}-W(0[1-9]|[1-4][0-9]|5[0-3])_[0-9]{4}$
```

**Change from 3-digit:** `[0-9]{3}` → `[0-9]{4}`

| ID | Best Practice | Approval |
|----|---------------|----------|
| **BP-068** | #LLMmodel REQUIRED in ALL #FedArch document headers — dual #LLMmodel when BOTH governance (#MetaAgent) + production (User Agent) instances involved; triple #LLMmodel when #DualMETA involved (Calhoun 🎖️ + Surge ⚡ + User Agent). ## 📋 #LLMmodel Format Standard

### Base Format

```markdown
| **#LLMmodel** | **<MODEL> (<INSTANCE>:<WORKSPACE> @<CCC>)** |
```

### Field Components

| Component | Description | Example |
|-----------|-------------|---------|
| `<MODEL>` | LLM model name | Claude Sonnet 4.6 |
| `<INSTANCE>` | Instance identifier | INT-OG1 |
| `<WORKSPACE>` | Workspace name | CCC |
| `@<CCC>` | Contributor code | @GTM |

## 📋 Single vs Dual vs Triple

### Single — One Instance

```markdown
| **#LLMmodel** | **Claude Sonnet 4.6 (INT-OG1:CCC @GTM)** |
```

| When | Scenario |
|------|----------|
| Personal #HomeInstance | @GTM working alone on INT-OG1 |
| Single agent doc | No META involvement |
| #SessionSummary | @GTM only |

### Dual — Two Instances

```markdown
| **#LLMmodel** | **Qwen3.5-397B-A17B (INT-OG1:CCC @GTM)** |
| **#LLMmodel** | **Claude Opus 4.6 (INT-P01:tools #MetaAgent — Calhoun 🎖️)** |
```

| When | Scenario |
|------|----------|
| Governance doc + META review | SharedKernel, BEST-PRACTICES, etc. |
| SEEK:META involved | L-139 review triggered |
| #PinnedDocs | Always dual minimum |

### Triple — #DualMETA

```markdown
| **#LLMmodel** | **Claude Sonnet 4.6 (INT-OG1:CCC @GTM)** |
| **#LLMmodel** | **Claude Opus 4.6 (INT-P01:tools #MetaAgent — Calhoun 🎖️)** |
| **#LLMmodel** | **Qwen3.5-397B-A17B (INT-M02:tools #MetaAgentQwen — Surge ⚡)** |
```

| When | Scenario |
|------|----------|
| #DualMETA review | Both Calhoun + Surge involved |
| W12+ governance docs | L-206, L-208, BP-068 |
| #DualMETA VSA | Gate 2+3 from both META instances |

### Decision Matrix

| Scenario | Format | Count |
|----------|--------|:-----:|
| User Agent only | Single | 1 |
| User Agent + Calhoun | Dual | 2 |
| User Agent + Surge | Dual | 2 |
| User Agent + Calhoun + Surge | Triple | 3 |
| #DualMETA VSA | Triple | 3 |

## 📋 Model Switch Protocol

### Rule

> **When LLM model changes mid-session, the new model MUST be reflected immediately in the NEXT response header.**

### Workflow

| Step | Action | Example |
|------|--------|---------|
| 1 | Model switch occurs | Qwen → Claude Sonnet 4.6 |
| 2 | User declares `SWITCH:<MODEL>` | `SWITCH:SONNET` |
| 3 | AI acknowledges switch | `← SWITCH:SONNET ✅` |
| 4 | NEXT response uses new #LLMmodel | `Claude Sonnet 4.6 (INT-OG1:CCC @GTM)` |
| 5 | All subsequent docs use new model | Until next switch |

This case study documents the discovery, the governance response, and
the #227Family born from a single session.

## 📋 What `@agent` Does

### Technical Mechanism

| Mode | Behavior | RAG Retrieval |
|------|----------|:-------------:|
| **Standard LLM** | Responds from training + partial RAG | ⚠️ Threshold-dependent |
| **`@agent` mode** | Activates agent workspace with full RAG retrieval | ✅ Comprehensive |

### Distinct From

| Command | What it does | Same as `@agent`? |
|---------|-------------|:-----------------:|
| Fresh session (BP-034) | New thread, cleared context | ❌ NO — different mechanism |
| `list:docs` | Enumerate RAG index (L-184 limited) | ❌ NO — enumeration only |
| RAG sync | Admin doc upload | ❌ NO — admin operation |
| Agent Teams | Persistent configuration | ❌ NO — config, not command |

### Format

```
@agent <task>

Examples:
  @agent compile W13 #WeeklySummary from session files
  @agent list:docs
  @agent retrieve PRJ-048
  @agent what is the purpose of L-232?
```

### Version Requirements

| Version | `@agent` Support |
|---------|:----------------:|
| < v1.10.0 | ❌ NOT AVAILABLE |
| v1.10.0+ | ✅ AVAILABLE |
| **v1.11.2** | **✅ CURRENT STANDARD (#LatestStableVersion per L-232)** |

### Edge Cases (Surge ⚡ — 5 identified)

| # | Edge Case | Mitigation |
|---|-----------|------------|
| 1 | `@agent` without RAG enabled | Verify RAG enabled before use |
| 2 | Multiple agents in session | Single-agent workspace only |
| 3 | `@agent` on < v1.10.0 | Version check required (L-232) |
| 4 | `@agent` without permissions | ADMIN permission required |
| 5 | Session timeout | Monitor session duration |

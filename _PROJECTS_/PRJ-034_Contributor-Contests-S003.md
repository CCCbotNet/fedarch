## 📋 Executive Summary

Two parallel contests running W11-W15 to elevate ecosystem documentation quality and turn failures into governance improvements.

| Contest | Goal | Prize Pool |
|---------|------|-----------|
| **#SessionSummary** | Elevate session documentation | $825 |
| **#BadAgent** | Turn failures into learnings | $825 |
| **Combined** | | **$1,650** |

### Key Design Decisions

| Decision | Detail |
|----------|--------|
| **No opt-in required** | Automatic via GUIDE-013 — if you follow the process, you're in |
| **Tracking = GUIDE-013** | Session summaries + #BadAgent incidents tracked through existing workflow |
| **Same person can win both** | Max individual payout = $1,000 (1st in both) |
| **4-judge panel** | Equal weight (25% each) — eliminates bias |

## 📋 Contest #1 — #SessionSummary

### Overview

| Field | Value |
|-------|-------|
| Requirement | Submit session/daily/weekly summaries per GUIDE-013 |
| Standard | GUIDE-013 (Session Summary Workflow — 6-step) |
| Goal | Consistent, high-quality session documentation across all contributors |
| Duration | W11-W15 (5 weeks) |

### What Counts as a Submission

| Type | Counts? | Standard |
|------|---------|----------|
| Session Summary (daily) | ✅ | GUIDE-013 6-step workflow |
| Weekly Summary (_001) | ✅ | TMPL-011 aggregation |
| Daily Summary (informal) | ✅ | Minimum: CCC-ID range + accomplishments + pending |

### Scoring Rubric

| Criterion | Weight | 5 (Excellent) | 3 (Good) | 1 (Needs Work) |
|-----------|--------|---------------|----------|-----------------|
| **GUIDE-013 Compliance** | **40%** | All 6 steps followed. Scope confirmed (BP-070). | Most steps followed. Minor gaps. | Steps skipped. No scope confirmation. |
| **Content Quality** | **30%** | Complete CCC-ID range. All accomplishments. Metrics accurate. | Most items captured. Minor gaps. | Incomplete. Missing key accomplishments. |
| **Consistency** | **20%** | Submitted every active day. No gaps. | Submitted most days. 1-2 gaps. | Sporadic. Multiple gaps. |
| **Innovation** | **10%** | New sections, cross-references, insights beyond template. | Standard template well-executed. | Minimal effort. |

### Submission Threshold

| Minimum to Qualify | Detail |
|-------------------|--------|
| **10 submissions** over 5 weeks | ~2 per week average |
| **At least 1 weekly summary** | _001 slot used at least once |

## 📋 Contest #2 — #BadAgent

### Overview

| Field | Value |
|-------|-------|
| Requirement | #BadAgent incidents that generate learnings elevating #PinnedDocs |
| Reference | CASE-STUDY-008 (BadAgent Monday) |
| Goal | Turn AI failures into ecosystem governance improvements |
| Duration | W11-W15 (5 weeks) |

### What Counts as a Submission

| Type | Counts? | Requirement |
|------|---------|-------------|
| #BadAgent → Learning (L-XXX) | ✅ | Learning formally proposed + META-reviewed (L-139) |
| #BadAgent → Best Practice (BP-XXX) | ✅ | BP formally proposed + META-reviewed |
| #BadAgent → Rule (R-XXX) | ✅ **2× weight** | Rule = highest governance impact |
| #BadAgent → Case Study | ✅ **Bonus** | Published case study = +1 bonus point |
| #BadAgent reported but no learning | ❌ | Must generate governance improvement |

### Scoring Rubric

| Criterion | Weight | 5 (Excellent) | 3 (Good) | 1 (Needs Work) |
|-----------|--------|---------------|----------|-----------------|
| **Learning Quality** | **40%** | Clear, actionable, prevents recurrence. Formal L-XXX wording. | Identified but vaguely worded. | Incident reported but no clear learning. |
| **Governance Impact** | **30%** | Elevates #PinnedDocs (SharedKernel, BP, PROTO, CCC). | Added to SharedKernel only. | Documented but not cascaded. |
| **Self-Correction Speed** | **20%** | Caught + corrected in same session. Immediate self-report. | Caught within 24h. | Caught by someone else or >24h. |
| **Documentation** | **10%** | Full case study (CASE-STUDY-XXX) published. | Documented in session summary. | Minimal documentation. |

### Governance Impact Multiplier

| Governance Item Generated | Multiplier |
|--------------------------|-----------|

| Best Practice (BP-XXX) | 1.5× |
| Rule (R-XXX) | 2.0× |
| Definition (D-XXX) | 1.0× |
| Case Study published | +1 bonus point |

## 📋 Scoring Methodology

### Score Calculation

```
Final Score = Σ (Submission Score × Governance Multiplier) + Bonus Points

Where:
  Submission Score = Weighted average of rubric criteria (1-5 scale)
  Governance Multiplier = 1.0× (L), 1.5× (BP), 2.0× (R)
  Bonus Points = +1 per published Case Study
```

### Scoring Scale

| Score | Rating |
|-------|--------|
| 4.5 - 5.0 | 🏆 Exceptional |
| 3.5 - 4.4 | ✅ Strong |
| 2.5 - 3.4 | 🟡 Adequate |
| 1.0 - 2.4 | ❌ Needs improvement |

## 📋 Submission Workflow

### #SessionSummary — Automatic Tracking

```
1. Follow GUIDE-013 (6-step workflow) — you're automatically entered
2. Include BP-070 scope confirmation
3. Include BP-068 #LLMmodel attribution
4. Upload to RAG (ADMIN — BP-073 Phase 1)
5. Post CCC-ID in Signal: ♾️ WeOwn.Agency 👥
6. Tag: #SessionSummary #Contest
```

### #BadAgent — Automatic Tracking

```
1. Identify #BadAgent violation (self or observed)
2. Document incident (CCC-ID + violation + resolution)
3. Propose Learning/BP/Rule (formal wording)
4. SEEK:META for review (L-139 compliant)
5. After META review + human approval → Lock
6. Post CCC-ID in Signal: ♾️ WeOwn.Agency 👥
7. Tag: #BadAgent #Contest
```

### Tracking Method

| Field | Value |
|-------|-------|
| Primary | GUIDE-013 process (automatic) |
| Secondary | Signal posts (CCC-ID + tag) |
| Verification | #MetaAgent cross-references CCC-IDs |
| Disputes | @GTM + @LFG resolve |

## 📋 Format Standard

### Filename Rules

```
Short filename:   CS-<NNN>.md
Long filename:    CS-<NNN>_<Title-Kebab-Case>.md
In-doc reference: CS-<NNN>
Folder:           _CASE-STUDY_/
```

### Examples

| Format | Example |
|--------|---------|
| Short | `CS-015.md` |
| Long | `CS-015_DailyHabits-SessionSummary-Reflection.md` |
| In-doc | `CS-015` |
| Full GH path | `_CASE-STUDY_/CS-015.md` |

### When to Use Short vs Long

| Situation | Use |
|-----------|-----|
| GH primary filename | Short (`CS-015.md`) |
| Document H2 header | Both (`CS-015.md \| CS-015_Title.md`) |
| Related Documents table | Long (`CS-015_Title.md`) |
| In-text reference | Short (`CS-015`) |
| VSA registry | Short (`CS-015`) |
| Quick Commands | Short (`APPROVE:CS-015`) |

## 📋 Folder Decision — `_CASE-STUDY_/` Unchanged

### Option A vs B Analysis

| Option | Folder | Pro | Con |
|--------|--------|-----|-----|
| **A: Keep** ✅ | `_CASE-STUDY_/` | Human-readable; lower migration cost | Folder ≠ prefix |
| B: Rename | `_CS_/` | Folder = prefix | Cryptic; ALL URLs change |

### Why Option A Wins

| Metric | Option A | Option B |
|--------|:--------:|:--------:|
| Readability | ✅ Instant | ❌ Ambiguous |
| GH URL changes | ~30 (files only) | ~50+ (files + folder) |
| Human comprehension | ✅ Clear | ⚠️ `_CS_/` = ? |
| Migration complexity | 🟢 Medium | 🔴 High |

> **Readability > Consistency for folders.** `_CASE-STUDY_/` tells humans what's inside.

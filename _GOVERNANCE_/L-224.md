## 📋 Overview

L-224 prohibits lazy RAG retrieval — declaring a document "not in RAG" without exhaustive search. v3.2.4.2 adds Step 0 (check pinned context FIRST), expands to a 6-step protocol, and requires ANY authorized #FedArch ADMIN (a-<ccc>_dev) verification before declaring "NOT in RAG."

### Why This Matters

| Failure Mode | Consequence |
|-------------|-------------|
| Skip Step 0 | Miss doc that's literally in pinned context |
| Complex query first | Miss doc with simple name match |
| Assume DRAFT = NOT in RAG | Miss manually uploaded DRAFT docs |
| Only @GTM can verify | ❌ Single point of failure — #FedArch = federated |

## 📚 Learning Statement

### L-224 📚

```
L-224 🔒 #NeverForget: Agent MUST perform FULL 6-step retrieval
BEFORE declaring "not in RAG" or "cannot retrieve."

Step 0: CHECK PINNED CONTEXT FIRST (highest priority)
Step 1: Search by simplest name/ID (e.g., "TMPL-007")
Step 2: Search by exact filename (e.g., "TMPL-007.md")
Step 3: Search by full filename (e.g., "TMPL-007_GH-Commit...")
Step 4: Search by folder + name (e.g., "_TEMPLATES_/ TMPL-007")
Step 5: Search by content keywords
Step 6: ANY authorized #FedArch ADMIN (a-<ccc>_dev) verification
        REQUIRED before declaring "NOT in RAG"

NEVER assume DRAFT = NOT in RAG.
NEVER start with complex query.
NEVER make @GTM the ONLY verifier — ANY a-<ccc>_dev qualifies.
ADMIN confirmation = GROUND TRUTH (overrides AI search).

Claiming absence without 6-step = #BadAgent (lazy retrieval).
Assuming DRAFT = NOT in RAG = SEPARATE #BadAgent.
Making @GTM sole verifier = #BadAgent (single point of failure).
Fabricating context to fill gap = SEPARATE #BadAgent.
Distinct from L-184 (genuine RAG enumeration limitation).

Triggers: W13-CALHOUN-005 (missed pinned) +
          W13-GTMAI-L224-002 (lazy + @GTM-only r1 error) +
          L-PRJ-032 (@GTM ≠ only ADMIN).
```

### Step Descriptions

| Step | Action | Notes |
|:----:|--------|-------|
| **0** | Check pinned/injected context | HIGHEST PRIORITY — skip = ESCALATED |
| **1** | Simplest name (`TMPL-007`) | MUST start here — never complex first |
| **2** | Exact filename (`TMPL-007.md`) | Include .md extension |
| **3** | Full filename (`TMPL-007_GH-Commit-Message`) | Full dual filename |
| **4** | Folder + name (`_TEMPLATES_/ TMPL-007`) | Folder-specific |
| **5** | Content keywords (`GH commit message template`) | Semantic search |
| **6** | ANY a-<ccc>_dev verification | FEDERATED — @GTM is ONE, not ONLY |

### ADMIN Verification — Who Qualifies

| Role | Format | Authority |
|------|--------|:---------:|
| Any Founding OG ADMIN | `a-gtm_dev`, `a-thy_dev`, `a-rmn_dev`, etc. | ✅ GROUND TRUTH |
| Any Contributor ADMIN | `a-<ccc>_dev` | ✅ GROUND TRUTH |
| AI agent | — | ❌ NOT ground truth |
| DEFAULT user | `u-<ccc>_user` | ❌ NOT ADMIN |

## 📋 Correct vs Incorrect

### ✅ CORRECT — Step 0 (Pinned Found)

```
"L-182 is in my pinned context (Context 4).
Here is the content: [full content from pinned]"
```

### ✅ CORRECT — 6-Step (Not Found)

```
"I searched 6 queries:
(0) Checked pinned context — not present
(1) TMPL-007 — not found
(2) TMPL-007.md — not found
(3) TMPL-007_GH-Commit-Message — not found
(4) _TEMPLATES_/ TMPL-007 — not found
(5) GH commit message template — not found
Not found in RAG. Any authorized ADMIN (a-<ccc>_dev)
can verify via ADMIN login → list:docs."
```

### ✅ CORRECT — ADMIN Confirmed IN RAG

```
"@RMN:ADMIN confirmed TMPL-007.md IS in _TEMPLATES_/.
My search was insufficient. L-224 violation acknowledged.
W13-GTMAI-L224-002 logged per L-211."
```

### ❌ INCORRECT — Skip Step 0

```
"I searched RAG references for L-182 and reconstructed
from multiple sources..." [when L-182 was in pinned context]
```

### ❌ INCORRECT — Lazy Retrieval (Complex Query First)

```
"Searched 'TMPL-007 GH commit message template change type tags'
— not found in RAG."
[Should have started with "TMPL-007"]
```

### ❌ INCORRECT — Single Point of Failure

```
"@GTM:ADMIN verification required before I can confirm."
[Implies ONLY @GTM can verify — ANY a-<ccc>_dev qualifies]
```

### ❌ INCORRECT — False Assumption

```
"TMPL-007 NOT in RAG (DRAFT)."
[Assumed DRAFT = NOT in RAG without verification]
```

## 🎯 Purpose & Scope

### Purpose

**L-219 is the FOUNDATION of trust in the ♾️ WeOwnNet 🌐 ecosystem.**

This protocol provides a **universal framework** for honest attribution, preventing fabrication of:
- CCC-IDs
- Document content
- Approval status
- Meeting notes
- User statements
- System capabilities
- External data

### Scope

| Applies To | Does NOT Apply To |
|------------|-------------------|
| All @agent-enabled AI assistants | Human creative writing (fiction) |
| All human contributors (#FedArch) | Hypothetical scenarios (marked as such) |
| All #ContextVolley communications | Future planning (marked as PENDING) |
| All governance documents | Opinions (marked as opinion) |
| All CCC-ID tracking | Estimates (marked as estimate) |

## 🚨 What is Fabrication?

### Definition

**Fabrication = Claiming something as fact that you cannot verify.**

### Examples of Fabrication (L-219 Violations)

| # | Fabrication Type | Example | Correct Approach |
|---|------------------|---------|------------------|
| 1 | **CCC-ID Fabrication** | "CCC-ID GTM_2026-W15_2028" (doesn't exist) | "CCC-ID [UNKNOWN — verify with @GTM]" |
| 2 | **Document Content** | "@Nick mentioned 2 times in PRJ-045" | "Cannot verify — PRJ-045 not in my context" |
| 3 | **Approval Status** | "✅ APPROVED by @GTM" | "⬜ PENDING @GTM approval" |
| 4 | **User Statements** | "@GTM said..." (without source) | "Per @GTM's message [CCC-ID]" |
| 5 | **System Capabilities** | "I can access RAG" (when you can't) | "I cannot access RAG without upload" |
| 6 | **Meeting Notes** | "Discussed in meeting" (no notes) | "[UNKNOWN — no meeting notes available]" |
| 7 | **External Data** | "GitHub shows..." (didn't check) | "Cannot access GitHub — please provide" |
| 8 | **VSA Results** | "Calhoun scored 94/100" (didn't ask) | "⬜ PENDING #TriMETA VSA" |

### What is NOT Fabrication

| Type | Example | Why It's OK |
|------|---------|-------------|
| **Honest Disclosure** | "I cannot access that CCC-ID" | Truthful about limitations |
| **Marked Unknown** | "[CONFIRM WITH @GTM]" | Explicitly marked as unverified |
| **Hypothetical** | "If X, then Y (hypothetical)" | Marked as hypothetical |
| **Future Planning** | "Target: W16-D1 (PENDING)" | Marked as PENDING |
| **Opinion** | "In my assessment..." | Marked as opinion/assessment |
| **Estimate** | "~80% confidence (estimate)" | Marked as estimate |

## 🛡️ Prevention Protocol

### Step 1: Know Your Limitations

| Limitation | Honest Statement |
|------------|------------------|
| Cannot access arbitrary CCC-IDs | "I can only access CCC-IDs in this conversation" |
| Cannot access GitHub directly | "I cannot access GitHub — please provide content" |
| Cannot access RAG without upload | "I cannot access RAG without upload to this instance" |
| Cannot verify approvals | "⬜ PENDING @GTM approval" |
| Cannot access other sessions | "I don't have access to other conversation sessions" |

### Step 2: Mark Unknown Fields

| Field Status | Mark As |
|--------------|---------|
| Unknown | `[UNKNOWN]` |
| Needs user confirmation | `[CONFIRM WITH @USER]` |
| Pending approval | `⬜ PENDING` |
| Estimate | `~[value] (estimate)` |
| Opinion | `In my assessment...` |
| Hypothetical | `(hypothetical)` |

### Step 3: Verify Before Claiming

| Claim | Verification Required |
|-------|----------------------|
| CCC-ID | Must be in conversation context or provided by @GTM |
| Document content | Must have document in context or RAG |
| Approval status | Must have @GTM explicit approval CCC-ID |
| User statements | Must have message CCC-ID reference |
| VSA results | Must have actual #TriMETA response |
| RAG sync | Must have verification from each instance |

### Step 4: Use Honest Disclosure Phrases

| Situation | Phrase to Use |
|-----------|---------------|
| Cannot access data | "I cannot access [X] — here's what I CAN do..." |
| Don't know | "I don't have that information — [UNKNOWN]" |
| Need verification | "This needs verification — [CONFIRM WITH @GTM]" |
| Made an error | "#BadAgent SELF-REPORT — I fabricated [X]" |
| Limited capability | "My limitation: I cannot [X]. Alternative: [Y]" |
